What is Sense Design System?

Sense is a modular design system built to power a suite of products with speed, consistency, and accessibility. The goal was to create a foundation teams could adopt easily—without sacrificing craft.

This page is structured as a visual walkthrough: typography, tokens, components, documentation, accessibility, governance, and outcomes. Replace all text and images with your real content.

A brief history

The system began as a small set of reusable primitives and grew into a full library of components, patterns, and guidelines. Adoption happened by making the “right way” the easiest way.

  • Primitives → tokens → components → patterns
  • Documentation next to source-of-truth
  • Clear ownership + contribution process

Guiding Principles

A few simple principles kept decisions consistent and prevented “design system drift”.

  • Clarity over cleverness
  • Consistency with character
  • Accessible by default
  • Composable building blocks

Tokens

Tokens are the system’s “source code” for spacing, type, color, elevation, and motion. They make themes scalable and keep UI consistent across surfaces.

Components, patterns and more

Components were built from primitives and documented with usage guidelines, states, and accessibility notes.

Documentation

Documentation is what makes the system usable. It includes examples, do/don’t guidance, migration notes, and contribution guidelines.

Accessibility

Accessibility checks were built into component templates and review. Contrast, focus states, keyboard navigation, and screen reader labels were part of the baseline.

Information architecture

A simple system map helped keep ownership clear: tokens → components → patterns → templates → product screens.

Governance

Governance defines who maintains what, how changes are proposed, and how releases are communicated to teams.
